🎯 我们的差异化方向

大厂产品（QClaw / ArkClaw）资源厚，但通常面向专业开发者，Skill 系统设计偏技术。

我们的机会在于：让"0 基础用户"也能轻松定制 AI 的工作方式，完成第一个属于自己的 Skill 时，产生"我也会用 AI"的成就感驱动留存与口碑传播。

📌 阶段目标三件事

本阶段（Q2）聚焦三件事，不盲目扩张：

① Skill 用户体验设计 — 从发现到使用，全链路低门槛
② 内容生态架构设计 — 内置库 + 用户自建 + 分享机制
③ 内置 Skill 库打磨 — 精选高频场景，打样板，建质量标准

Section 02

我们必须回答的核心问题

以下是在正式动工前，团队必须通过调研 + 讨论得出结论的关键决策点。

  • 1
    我们的目标用户到底是谁？他们用 Skill 解决什么问题？
    下沉市场宽泛，需要明确是"小白职场用户"还是"非技术创业者"还是"学生群体"，用户画像不同，Skill 内容和交互方式天差地别。
  • 2
    Skill 的最小可理解定义是什么？用户说得出来吗？
    如果用户无法用一句话说明白"Skill 是什么"，我们的概念设计就失败了。需要在产品语言层面找到非技术用户能接受的比喻（如"AI 的工作习惯"、"助手的性格设定"）。
  • 3
    用户自建 Skill 的门槛底线在哪？最简流程是几步？
    成就感驱动的核心是"第一次成功"足够快。如果自建流程超过 5 步或出现任何技术词汇，用户大概率放弃。需要明确最简可行流程（MVP 自建路径）。
  • 4
    内置 Skill 库的选品标准是什么？做多少个算够？
    大而全不如小而精。需要确定内置库的选品原则：高频场景优先？还是能展示差异化？并定义"打磨完成"的质量标准。
  • 5
    Skill 分享/传播的激励机制如何设计？
    生态冷启动最难。"用户创建 → 分享 → 他人使用 → 创作者获得反馈"的正循环能否跑起来，取决于传播激励和成就感设计，需要明确玩法。
  • 6
    竞品 Skill 体系的核心差距在哪？我们能超越的点是什么？
    调研不只是"他们有什么"，更要找"他们哪里做得烂/下沉用户在哪里被劝退"，这是我们的机会点。

Section 04

团队角色分工

基于三人的角色定位，明确每人的核心职责与能力成长方向，AI 工具作为每人的协作增强器。

你（Leader）
策略把控 · 体验主导 · 对齐研发
  • Skill 整体产品定义与边界划定
  • Skill 中心信息架构与入口设计（B1）
  • 自建引导核心流程设计（B3）
  • 内置库选品策略拍板（C1）
  • 内容质量标准制定（C2）
  • 与研发对齐技术约束（数据结构 / 触发机制）
  • 阶段性成果 Review + 方向校准
员工 A
竞品研究 · 交互体验 · 内容生产
  • 竞品 Skill 体系深度拆解（A1）
  • Skill 使用全链路 UX 旅程设计（B2）
  • Skill 管理页交互设计（B4）
  • 首批内置 Skill 内容撰写与测试（D1/D2）
  • 协助整理竞品调研结论汇报材料
员工 B
用户研究 · 生态运营 · 内容包装
  • 目标用户访谈与画像输出（A2）
  • 概念认知测试（A3）
  • 自建 Skill 内容框架设计（C3）
  • 分享与冷启动方案（C4）
  • 成就感激励体系协同设计（B5）
  • 内置 Skill 展示文案与封面（D3）
AI 工具使用建议：两位员工虽无技术背景，竞品分析可用 AI 辅助结构化整理；用户访谈可用 AI 辅助设计访谈提纲和分析总结；内置 Skill 内容撰写可用 AI 辅助 Prompt 调优和效果测试；Figma + AI 工具可覆盖交互设计原型输出需求。
Section 05

工作推进节奏

分四个阶段推进，每阶段结束有明确可交付产出用于内部对齐，不设时间 deadline，以交付物完成度为推进信号。

  • P1

    阶段一：认知底座建立

    完成竞品拆解（A1）+ 用户访谈（A2）+ 产品概念测试（A3）。
    对齐产出：「竞品 Skill 体系对比报告」+「目标用户画像 & 需求洞察」。
    全团队看完报告，统一认知后进入设计阶段。

  • P2

    阶段二：核心体验设计

    完成 Skill 发现/入口设计（B1）+ 自建引导流程（B3）+ 使用链路 UX（B2）。
    对齐产出：「Skill UX 设计方案 v1」（包含信息架构 + 关键流程原型）。
    内部走查后确认可以开展内容工作。

  • P3

    阶段三：内容策略 & 生态架构

    完成选品策略（C1）+ 质量标准（C2）+ 自建内容框架（C3）+ 分享冷启动方案（C4）。
    对齐产出：「内置 Skill 库规划书」+「Skill 生态架构设计方案」。

  • P4

    阶段四：内置库首批内容生产

    完成首批 8~12 个内置 Skill 的内容撰写（D1）+ 效果测试（D2）+ 展示包装（D3）+ 管理页设计（B4）+ 成就感激励方案（B5）。
    对齐产出：「Skill 模块完整交付包」，可进入研发提审。

Section 06

需要提前关注的风险

提前识别，不是让你焦虑，是让你有预案。

⚠️ 风险 1：用户画像失准
"下沉市场"是个宽泛标签，如果访谈样本过于集中（如全是同学/朋友），很容易得出不真实的用户需求。建议员工 B 提前设计招募标准，确保受访者真实符合目标画像（如：非技术职业 + 正在用 AI 工具 + 非一线城市）。
⚠️ 风险 2：自建流程陷入"功能堆叠"
低门槛设计最大的陷阱是设计者高估用户能力，在"简易"路径里塞太多选项。建议 B3 设计完成后，找 3 个真实目标用户做冷启动测试，观察他们能否在 5 分钟内独立完成第一个 Skill 创建。
⚠️ 风险 3：内置库选品与实际使用脱节
内置 Skill 如果基于"我们觉得用户需要什么"而非真实数据选品，上线后大概率使用率惨淡。C1 选品阶段必须以用户访谈结论（A2）作为输入，避免拍脑袋。
📌 建议预留的对齐节点
每阶段结束后，建议与研发团队做一次简短的技术可行性确认会（≤1 小时），确保 UX 方案和生态设计中的关键机制（如 Skill 触发条件、上下文注入方式）在技术上可落地，避免设计完发现做不了的情况。
