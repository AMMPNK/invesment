CSIG 是什么

CSIG全称「云与智慧产业事业群」，是腾讯面向B端企业的核心业务部门，旗下涵盖腾讯云、腾讯会议、腾讯文档、微信支付B端等产品线。计费团队属于腾讯云基础服务层，为整个CSIG的所有云产品提供统一的计费系统支撑。

腾讯云 企业服务 基础设施 支撑型产品
计费团队职责

面试官透露团队负责CSIG计费系统规划，核心四大模块：

交易管理 → 资源管理 → 优惠折扣 → 账票款

本质：让企业客户能清晰地购买、使用、核算云资源费用，并享受各类优惠政策。

腾讯云费用中心 — 产品全景
账单体系
  • 账单概览：总消费汇总、趋势图
  • 多维汇总账单：按产品/项目/地域聚合
  • 资源账单：单个资源粒度消费
  • 明细账单：每笔交易流水
  • 账单订阅：定时推送邮件/Excel
优惠折扣体系
  • 代金券：满额抵扣，有面值限制
  • 折扣券：按比例打折，不与代金券叠加
  • 资源包：预购特定用量，比按量更优惠
  • 长期折扣：按量计费自动阶梯折扣
  • 包年包月：预付费，享受买断优惠
交易 / 支付体系
  • 预付费：包年包月，一次性付清
  • 后付费：按量计费，月结账单
  • 冻结机制：按量计费时预冻结余额
  • 退款规则：5天无理由退/按比例退
  • 集团账号：管理账号代付成员账号
账票款管理
  • 发票管理：增值税普票/专票申请
  • 合同管理：框架合同、订单合同
  • 对账指引：与ERP系统对账流程
  • 成本分配：按业务线/项目分摊费用
  • 欠费处理：欠费停服 → 回收 → 销毁

三轮面试流程拆解

基于牛客网 18 篇 CSIG 产品策划真实面经整理，了解每轮考察重点

一面 — Team Leader 直属面试

时长约 50–60 分钟 · 产品能力 + 项目深度 · 难度最高
最重要

TL 会深度挖掘简历，结合 B 端计费业务设计场景题，期望看到结构化思维 + 数据意识 + 产品感

Q详细介绍你做过的某个产品项目，你在其中承担什么角色？遇到了哪些困难？
必问题，用 STAR 框架：背景 → 任务 → 行动 → 结果，重点量化成果
Q假设你是腾讯云费用中心 PM，发现某大客户月账单明细复杂，如何设计优化方案？
产品设计题，先定义「谁的问题」「什么场景」「什么痛点」再给方案
Q如果代金券使用率突然下降 30%，你会如何定位问题？
数据分析题，展示：拆解指标 → 提出假设 → 设计验证路径
参考思路：先区分是「发放量下降」还是「领取率下降」还是「核销率下降」→ 分渠道分客群看数据 → 排查是产品 bug / 规则变更 / 竞品优惠更大 → 用 AB 测试验证假设
Q微信有 5 个常用功能，如果只能保留 1 个，你选哪个？说出理由。高频
来自多篇 CSIG 真实面经，考察产品核心价值理解 + 用户需求洞察
参考框架：选「通讯/聊天」功能。① 用户核心诉求是「沟通连接」，这是微信立身之本；② 其他功能（朋友圈/支付/小程序）都是在连接能力上的延伸，失去连接则平台价值归零；③ B 端延伸：企业微信场景下「群聊+文件传输」比朋友圈更核心
Q腾讯会议 DAU 疫情期间暴涨，疫情后如何分析并保留用户？
分析用户生命周期变化，给出差异化留存策略
参考框架：① 区分疫情期「刚需用户」和「习惯用户」② 流失预测：哪类用户在哪个节点最可能流失③ 功能差异化：「远程开会」→「云协作办公」④ 与企业微信联动，嵌入 B 端工作流，提升替换成本
Q如何判断一个新功能需求是否值得做？你会从哪些维度评估优先级？
可以提 RICE 模型（覆盖面/影响力/置信度/工作量）
Q费米问题：估算北京市一天产生多少快递包裹？高频
来自多篇 CSIG 面经，考察结构化拆解思维，过程比答案更重要
答题模板：① 假设：北京约 2000 万人口，上班族/学生约 1200 万人 ② 频率：平均每人每周网购 2 次，约 1 件/次 ③ 计算：1200 万 × 2/7 ≈ 340 万件/天，约 300–400 万件 ④ 验证合理性：对比双 11 峰值数据

二面 — 总监面

时长约 15–20 分钟 · 业务认知 + 战略视角 · 云计算深度
高风险
真实面经预警（来自牛客网 CSIG 面经）

总监面「说一说你对云计算的了解」——某应聘者只被问了 5 个问题，不到 15 分钟就结束面试，最终因对云计算了解深度不够而凉。另有前腾讯云实习生被评价「拥有 PM 必备技能，但对云计算了解不够专精」。
→ 云计算专业深度是 CSIG 二面核心考察点，必须重点准备！

Q说一说你对云计算的了解。必问高危
总监级面试官问完就决定要不要你，答不好直接凉
黄金答题框架（3 层递进）：
① 基础认知：IaaS/PaaS/SaaS 三层架构，公有云/私有云/混合云；云计算本质是「IT 资源的按需租用」，帮企业将 CAPEX 转为 OPEX
② 行业理解：当前云计算处于「规模化扩张期」，国内三大云（阿里/腾讯/华为）竞争激烈，向行业云/边缘计算/AI 云演进；计费系统是云商业化的核心底座
③ 业务洞察：腾讯云 CSIG 面向 B 端，计费系统核心价值是「让企业清晰买、用、算云资源费用」，FinOps 是未来重要方向
Q怎么看待云计算行业现在发展到什么阶段，今后向什么方向发展？
来自真实字节云产品面经，腾讯云实习背景者被深问
参考框架：① 成熟期：IaaS 竞争红海化，价格战进入尾声 ② 向上走：PaaS/SaaS/AI 云差异化 ③ 向下走：边缘计算、IoT 云延伸 ④ FinOps 兴起：企业上云后重视成本治理，云管理平台成新战场
Q你做过的哪个项目，认为有没有做到极致？如果重来一次会怎么做？
复盘能力题，展示自我迭代的思考，不要只说优点
Q竞品（阿里云/AWS）的计费体系相比腾讯云有什么值得借鉴的地方？
提前研究阿里云费用中心和 AWS Cost Explorer 的设计差异
参考要点：AWS Cost Explorer 标签体系成熟，支持自定义成本分摊；阿里云账单可视化更好；腾讯云可强化企业级成本归因和 FinOps 主动优化建议
Q如果你来做腾讯云计费系统的产品规划，第一优先级是什么方向？
展示对业务的理解深度，不要随便说，先做功课

时长约 20–30 分钟 · 价值观 + 沟通能力 + 稳定性
几乎必过

高频题目 + 答题框架

结合你的快手 AI 直播实习背景，定制化的答题模板

1自我介绍（100% 出现）
STAR 式自我介绍框架 · 建议 3 分钟
①背景：北邮+中传双校研一，AIGC 方向，研究方向是[补充]
②亮点项目：快手女娲数字人 AI 直播产品实习，参与[具体功能]，数据结果是[量化数字]
③技能匹配：有 ToC 产品经验 + 数据分析能力（Python/SQL/Tushare），现在想向 ToB 转型
④兴趣切入：对云计算计费系统感兴趣，调研过腾讯云费用中心 vs 阿里云 vs AWS 的计费产品设计
⑤期待收获：期望深入学习 B 端产品设计方法论和云计算业务逻辑
关键技巧：介绍快手经历时埋一个「钩子」，比如「在快手实习时我注意到 AI 功能的内部计费逻辑和 ToB 系统有很多相似之处，这让我对企业级计费系统产生了浓厚兴趣」——能自然引导面试官往你准备好的方向追问。
2项目经历深挖（必问）
STAR 答题框架
S背景：项目在什么业务背景下启动？公司的目标是什么？
T任务：你个人负责哪部分？你的角色和职责边界是什么？
A行动：你具体做了什么？遇到了什么挑战？如何解决的？（最重要）
R结果：量化结果是什么？用户反馈如何？对业务的影响？有没有复盘？

一定要有量化数字：如「功能上线后 DAU 提升 X%」「用户完成率从 X% 提升至 Y%」「节省人工审核成本 X 元/天」

3数据分析题（计费团队必问）

计费系统本质是数据密集型产品，面试官一定会考察你的数据分析思维。

数据异常分析 通用框架
①确认数据：异常是真实的还是统计口径问题？数据源是否可信？
②拆解指标：按维度分解（时间/地区/用户类型/产品线），定位问题在哪个细分层
③提出假设：枚举所有可能原因（内部：功能上线/规则变更；外部：竞品/政策/季节性）
④验证假设：通过 SQL 查询、AB 实验、用户访谈等方式逐一排查
⑤给出结论：明确根因，给出可执行的修复/优化方案
4产品设计题 — 以计费场景为例
题目示例：「企业财务人员反馈腾讯云账单太复杂看不懂，如何优化账单产品？」
产品设计题答题框架
①用户洞察：财务人员 vs IT 采购人员 vs 业务负责人，各自需求不同，先明确目标用户
②场景还原：财务人员什么时候看账单？月末结账、对账 ERP、申请报销、向领导汇报
③痛点定义：复杂在哪里？SKU 太多/计量单位不一致/无法按部门拆分/缺少趋势对比
④方案设计：优先级最高的方案 → 次优方案，说明取舍理由
⑤成功指标：账单查看时长↓、投诉率↓、完成对账任务成功率↑
可以提到 FinOps（云财务运营）理念：让财务、业务、技术三方都能看懂云费用，这是当下云厂商的热门方向，能体现你的行业洞察。

云计费系统核心知识

面试时能主动提及这些概念，会显著提升专业印象

计费模式核心概念
  • 预付费（包年包月）：先付后用，折扣大，适合稳定业务
  • 后付费（按量计费）：用多少付多少，按小时/天结算，欠费冻结机制
  • 资源包：预购特定用量，比按量便宜但需提前规划
  • 竞价实例：大幅折扣但随时可能被回收，适合弹性任务
  • 账单周期：按量计费每小时出账，月底汇总账单
企业账号管理体系
  • 集团账号：管理账号 + 成员账号，统一计费
  • 代付模式：管理账号代成员账号支付费用
  • 账单归集：集团统一查看所有成员账单
  • 配额管理：管理账号可限制成员账号的资源配额
  • 财务隔离：不同业务线独立核算成本
优惠折扣体系设计
  • 代金券：有面值上限，用于拉新促活，不可提现
  • 折扣券：按比例打折，不与代金券叠加
  • 长期折扣：连续按量计费自动享受阶梯优惠
  • 商务折扣：大客户签合同可享受定制化折扣
  • 欠费规则：欠费时不可使用代金券新购
FinOps 云财务管理
  • FinOps 理念：让财务+技术+业务三方共同管理云成本
  • 成本归因：通过标签 Tag 将费用归属到具体项目/部门
  • 成本优化：识别浪费资源（闲置实例、低使用率资源）
  • 预算预警：设置消费阈值，超额自动通知
  • 成本预测：基于历史用量预测未来费用
竞品速览 — 腾讯云 vs 阿里云 vs AWS 计费产品
对比维度 腾讯云费用中心 阿里云费用中心 AWS Cost Explorer
账单可视化基础图表，产品分类较完善，多维度视图最强，自定义图表
成本分配项目/标签维度资源组+标签成本分配标签最成熟
企业账号集团账号体系企业账号AWS Organizations
成本优化建议基础提示云优化顾问Cost Optimization Hub
FinOps 支持成长中云财管 CFO行业最成熟
可以在总监面主动提出：腾讯云在 FinOps 方向还有很大提升空间，可以参考 AWS Cost Optimization Hub 的「AI 智能推荐闲置资源」功能，结合腾讯在 AI 方面的优势，为企业客户提供主动式成本优化建议。

B端产品核心方法论

从你的 ToC 快手经历向 ToB 平滑过渡，展示转型的思考

ToC vs ToB 核心差异（背下来）
维度 ToC（你的经历：快手） ToB（目标：腾讯云计费）
目标用户个人用户，情感/娱乐驱动企业客户，效率/ROI 驱动
决策链路用户自己决策，路径短财务+IT+业务负责人多方审批
核心价值好玩、好看、好用降本、增效、合规
反馈节奏快速迭代，AB 测试验证需求定制，周期长，重服务
盈利模式广告+增值服务+电商直接收费/订阅/按量付费
产品失败成本用户流失，可快速挽回客户流失，合同违约，损失巨大
转型加分话术：「我在快手实习的经历培养了我对用户体验的敏锐感知，我意识到 ToB 产品同样需要用户体验思维，只是『用户』变成了企业里的财务专员、IT 管理员等不同角色，需要同时满足他们各自的核心诉求，这让我更想深入 ToB 领域。」
B端需求优先级 RICE 模型
  • Reach（覆盖度）：这个需求影响多少客户/用户？
  • Impact（影响力）：解决这个问题的价值有多大？
  • Confidence（置信度）：我们有多确定这个方案有效？
  • Effort（工作量）：需要投入多少研发成本？
RICE 得分 = (R × I × C) / E
B端需求调研方法
  • 主动获取：竞品分析、行业调研、问卷调查
  • 被动收集：客户反馈、客服工单、销售传递
  • 需求过滤：区分「个性需求」和「共性需求」
  • 价值验证：增收/降本/提效，量化业务价值
  • MVP 验证：最小成本验证核心假设
B端产品成功指标体系
商业指标
  • ARR 年经常性收入
  • NRR 净收入留存率
  • LTV/CAC 客户价值比
  • 客户流失率 <20%
产品指标
  • 功能使用率/渗透率
  • 任务完成率
  • 账单查询成功率
  • 客户满意度 CSAT
效率指标
  • 账单生成时延
  • 对账准确率
  • 客服投诉率
  • 发票开具时效
